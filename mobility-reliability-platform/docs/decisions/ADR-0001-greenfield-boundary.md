# ADR-0001: 신규 코드베이스와 기존 자산 경계

- 상태: accepted
- 결정일: 2026-07-21

## 맥락

기존 프로젝트는 기업 협업 과정에서 사용한 상표와 제품 자산 때문에 같은 형태로 계속 발전시키기 어렵다. 세 기존 저장소 역시 웹, 관리자, MongoDB 백엔드, Firebase 전환 상태가 혼재한다.

## 검토한 선택지

1. 기존 저장소를 리브랜딩하며 점진적으로 수정
2. 기존 코드를 포크해 새 UI만 구축
3. 신규 저장소를 만들고 도메인 구조와 승인된 데이터만 참고

## 결정

선택지 3을 채택한다.

- 기존 코드·UI·브랜드·문구를 신규 runtime에 복사하지 않는다.
- 기존 DB 형식과 수리데이터는 사용할 수 있다.
- 데이터는 별도 importer와 품질 게이트를 거쳐 신규 ID·이벤트 구조로 변환한다.
- 기존 Firebase나 MongoDB를 신규 서비스의 runtime dependency로 만들지 않는다.

## 결과

- 초기 구현 비용은 늘지만 상표·기술부채·혼합 아키텍처에서 분리된다.
- 기존과의 기능 동등성보다 새로운 데이터 계약과 모바일 텔레메트리가 우선된다.
- 이관 가능한 데이터 범위는 코드 재사용 범위와 별도로 추적한다.
