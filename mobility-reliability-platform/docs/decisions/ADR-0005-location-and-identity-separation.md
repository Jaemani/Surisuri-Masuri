# ADR-0005: 위치·신원 데이터 분리와 생명주기

- 상태: accepted
- 결정일: 2026-07-21

## 맥락

정밀 위치와 장애·복지 정보의 결합은 사용자에게 큰 위험이 될 수 있다. event sourcing을 이유로 삭제 대상 개인정보를 영구 보관해서는 안 된다.

## 결정

- 인증 신원과 도메인 actor ID를 분리한다.
- 위치 샘플은 pseudonymous device/session ID만 가진다.
- 원본 좌표에는 별도 TTL과 사용자 삭제 절차를 적용한다.
- 감사 대상 메타데이터와 삭제 대상 payload를 분리한다.
- 복지관 콘솔은 기본적으로 원본 경로를 표시하지 않는다.
- 지역 집계는 소수 사용자가 드러나는 셀을 억제한다.
- tenant는 인증된 membership에서 결정하며 클라이언트 필드를 신뢰하지 않는다.

## 결과

- 일부 원본이 삭제되어도 receipt와 처리 사실을 증명할 최소 감사정보가 남는다.
- 파생 데이터는 원본 삭제와의 관계 및 재식별 위험을 별도로 관리해야 한다.
- 보존기간의 구체 값은 실증 동의서와 기관 정책 확정 후 기록한다.
