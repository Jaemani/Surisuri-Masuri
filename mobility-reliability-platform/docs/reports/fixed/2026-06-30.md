## 목적

연결이 불안정한 현장에서도 이벤트가 유실되지 않도록 offline-first 동기화 규약을 정의한다. 배터리와 데이터 품질을 함께 다루는 적응형 샘플링의 기준선도 마련한다.

## 계획상 논의

- SQLite append-only 큐, 업로드 배치, acknowledgment, 재시도 상태
- 6월 검증은 결정론적 local/mock sync adapter를 사용하고, 실제 Cloud Run·Firestore 반영은 R05/R06에서 다시 검증하는 단계 경계
- client sample/session/batch correlation ID와 서버 파생 idempotency key를 이용한 중복 방지
- 순서가 뒤바뀐 이벤트, 부분 성공, 앱 재설치, 장시간 offline 시 처리
- 서버 시간을 신뢰하지 않고 발생 시각과 수집 시각을 분리하는 규칙
- 정지·이동·낮은 정확도 상태에 따른 적응형 샘플링 후보
- 사용자가 원본 위치를 삭제하거나 동의를 철회할 때 로컬·서버 처리

## 결정 예정 항목

- SQLite 테이블과 동기화 상태 값
- 재시도 backoff, 최대 배치 크기, 보존 기간
- 같은 배치를 반복 전송했을 때의 local/mock adapter 응답 계약과 R05 production adapter 재검증 조건
- 적응형 샘플링 v0 규칙과 안전한 fallback
- 삭제·동의 철회 이벤트의 우선순위

## 기대 산출물

- offline sync 프로토콜과 시퀀스 다이어그램
- 비행기 모드 수집 후 재연결 데모
- local/mock sync adapter 기반 동일 배치 반복 전송·부분 실패 자동 테스트
- 고정 샘플링과 적응형 샘플링 비교표
- 모바일 텔레메트리 데이터 계약 v0.2

## 검증 지표

- 동일 이벤트를 10회 재전송해도 local/mock adapter의 논리 반영은 1회이며, production 반영 검증이 R05/R06으로 명시적으로 이관되었는가
- 앱 재시작 후 미전송 큐와 상태가 보존되는가
- 네트워크 복구 후 누락·중복 수가 자동 산출되는가
- 적응형 정책이 정의된 정확도 기준을 유지하면서 샘플 수 또는 배터리 소비를 줄이는가
