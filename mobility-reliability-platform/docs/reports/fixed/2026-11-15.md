## 목적

새 브랜드와 코드로 Firebase Hosting 기반 기관 콘솔을 구성하고, Firestore current projection·근거 리포트와 사용자·수리사 모바일 흐름을 실제 운영 가능한 pilot 절차로 연결한다. 고령자·장애인 접근성과 기관별 권한 격리를 기능 완료 조건에 포함한다.

## 계획상 논의

- 기관 콘솔의 Firestore 기기 상태 projection, 수리 타임라인, 점검 위험, 근거 리포트, 조건부 익명 공간 집계
- 기관별 출력 양식을 core model과 분리하는 schema adapter
- Firebase Auth/App Check, Firestore membership·Security Rules를 이용한 사용자·수리사·기관 관리자 최소 권한과 감사 로그
- pilot 참여·위치정보 동의·철회·데이터 삭제·문의 대응 흐름
- 큰 글씨, 명확한 수집 상태, screen reader, 색상 외 상태 표현, 터치 영역
- 과거 참여신청서에 제공된 50명 계획치(로컬 `attached` 참고, Git 미포함)는 원문·승인 상태를 사람 검토한 뒤에만 공식 KPI로 사용하고, 실제 모집·완료 수와 분리하는 원칙
