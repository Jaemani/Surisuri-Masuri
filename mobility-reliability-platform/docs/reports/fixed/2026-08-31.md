## 목적

학습한 데이터 품질 모델을 ONNX로 변환해 모바일에서 실행하고, 서버 추론 없이도 수집 품질 플래그를 생성할 수 있는지 검증한다. 기기별 성능 차이와 모델 fallback까지 포함해 배포 가능성을 판단한다.

## 계획상 논의

- PyTorch 체크포인트에서 ONNX export와 수치 일치 검증
- feature extraction을 Python과 모바일에서 동일하게 재현하는 계약
- float 모델과 quantized 모델의 크기·지연·정확도 비교
- Android/iPhone별 cold start, p50/p95 추론 지연, 메모리 사용량
- 입력 누락·비정상 길이·지원하지 않는 모델 버전의 fallback
- 모델 출력이 사용자 안전 판단이 아닌 데이터 품질 보조임을 UI에 반영

## 결정 예정 항목

- 배포 모델 구조와 모델·feature schema 버전
- 양자화 채택 여부와 허용 성능 저하
- inference window 길이와 실행 빈도
- on-device 결과를 서버에 저장할 최소 정보
- 모델 다운로드·rollback 또는 앱 내장 정책

## 기대 산출물

- ONNX 모델과 모델 카드 초안
- Android/iPhone 온디바이스 추론 데모
- PyTorch/ONNX 출력 parity 테스트
- 양자화 전후 정확도·크기·지연시간 표
- 모델 오류·fallback 진단 화면 또는 로그

## 검증 지표

- 동일 입력에서 PyTorch와 ONNX 출력 차이가 정한 허용오차 이내인가
- 두 플랫폼에서 목표 지연시간과 메모리 한도를 만족하는가
- quantization 이후 class별 성능과 calibration 변화를 측정했는가
- 모델이 실행되지 않아도 GPS 원본 수집·동기화가 중단되지 않는가
