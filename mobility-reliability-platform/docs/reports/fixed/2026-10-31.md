# 2026년 10월 2차 기술 정기리포트

> 문서 상태: 계획 기준 사전작성본  
> 실제 증빙: 회의 후 갱신

## 목적

사용자·수리사·복지관 리포트를 생성하되 모든 핵심 주장에 검증 가능한 Fact ID를 연결하는 AI report agent를 설계한다. 근거가 없거나 스키마를 위반한 문장은 차단·수정·유보하도록 실행 루프를 구성한다.

## 계획상 논의

- 통계·모델 출력을 정규화하는 Fact Store와 fact provenance
- 대상별 report JSON Schema 및 허용되는 주장 유형
- planner→writer→validator→fallback으로 구성한 실행 상태
- claim–evidence entailment, 수치 일치, 날짜 범위, 금지 표현 검사
- 모델/API 실패, timeout, invalid JSON, 근거 부족의 실패 분류
- 프롬프트·모델·도구 호출·검증 결과를 재현 가능하게 기록하는 receipt

## 결정 예정 항목

- Fact ID 형식과 불변성·버전 규칙
- 대상별 필수·선택 보고서 섹션
- 규칙 기반 validator와 모델 기반 validator의 경계
- 근거 없는 주장 발견 시 삭제·재생성·확인 필요 처리
- 에이전트 평가셋과 배포 승인 지표

## 기대 산출물

- Fact Store 스키마와 샘플 fact bundle
- report agent 실행 상태도와 provenance receipt
- 사용자·수리사·복지관 리포트 예시
- 공격·결측·모순 입력 평가셋
- claim–evidence 검사 결과 및 fallback 데모

## 검증 지표

- 모든 주요 수치·권장 문장에 유효한 Fact ID가 존재하는가
- 근거 없는 문장·수치 변조·오래된 데이터가 평가셋에서 탐지되는가
- 같은 입력과 버전 정보로 실행 경로를 재현할 수 있는가
- LLM 장애 시 핵심 사실을 규칙 기반 보고서로 제공할 수 있는가
