# 2026년 5월 2차 기술 정기리포트

> 문서 상태: 계획 기준 사전작성본  
> 실제 증빙: 회의 후 갱신

## 목적

웹 화면의 단순 포팅이 아닌 모바일 고유 기능을 중심으로 첫 end-to-end GPS 수집 경로를 설계한다. 스마트폰 이동과 전동보장구 주행을 혼동하지 않도록 명시적 주행 세션을 초기 기준으로 삼는다.

## 계획상 논의

- React Native 기반 신규 앱의 사용자·수리사 화면 경계
- Android와 iOS의 위치 권한, foreground/background 상태, 앱 종료 상태 차이
- 주행 시작·종료, 위치 샘플, 정확도, 속도, 시간, 기기 식별자를 포함한 이벤트 스키마
- 상시 수집 대신 명시적 주행 세션을 기본값으로 두는 이유
- 앱 내 원본 이벤트 임시 저장과 서버 전송 전 개인정보 최소화
- 지도 재생, 수집 상태, 권한 안내를 포함한 발표 가능한 vertical slice

## 결정 예정 항목

- 모바일 프레임워크와 native module 경계
- 최초 지원 OS 버전 및 테스트 기기 행렬
- 초기 GPS 수집 간격·정확도 임계값의 실험 범위
- 주행 세션의 시작·중지·중단 복구 상태 머신
- 외부 GPS/IoT 센서는 배제하고 모바일 GPS만 사용하는 데이터 계약

## 기대 산출물

- 모바일 앱 셸 및 새 디자인 토큰 초안
- 위치 권한과 주행 세션 상태도
- 위치 이벤트 JSON Schema v0.1
- 단일 테스트 경로를 지도에서 재생하는 PoC
- ADR: 자체 모바일 GPS를 1차 데이터원으로 채택

## 검증 지표

- 두 플랫폼에서 권한 상태별 UI가 구분되는가
- 각 이벤트에 고유 ID와 발생 시각·수집 시각이 분리되는가
- 테스트 경로가 순서와 정확도 정보를 보존해 재생되는가
- 사용자가 수집 중 여부와 중지 방법을 명확히 인지할 수 있는가
