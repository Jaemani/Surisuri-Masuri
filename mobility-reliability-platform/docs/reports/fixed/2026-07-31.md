## 목적

Cloud Storage의 압축 위치 batch가 정제·요약되고 Firestore receipt와 current projection으로 연결되는 전 과정을 프라이버시·비용·복구 관점에서 검증한다. BigQuery GIS/ML은 기본 구성으로 간주하지 않고 실제 분석 요구와 비용 게이트를 통과할 때만 활성화 후보로 다룬다.

## 계획상 논의

- 비정상 속도, 순간 점프, 낮은 정확도, 장시간 정지를 다루는 Storage batch projector 규칙
- 원본 Storage object, Firestore receipt, trip summary, current projection의 별도 보존 기간과 접근 역할
- 출발·도착 반경 마스킹, 공간 해상도, 최소 집계 인원 기준
- Storage lifecycle과 사용자 삭제 시 Firestore metadata, Storage object, optional BigQuery 파생 데이터까지 추적하는 deletion job
- Firestore realtime listener를 운영상 필요한 작은 projection에만 제한하는 비용 원칙
- Cloud Run 처리 지연·Storage 오류·Firestore receipt/projector 실패 시 재처리와 관측 항목
- pending receipt의 단일 active lease owner와 단조 증가 fencing token, stale finalizer 차단 규칙
- raw-only, no-artifact, manifest-only, stored-missing 상태를 구분하는 crash recovery matrix
- lease 만료 전 forward reconciliation과 만료 후 generation-pinned cleanup target, deadline cleanup 전환 경쟁
- BigQuery 도입 전 승인된 Storage manifest를 ML 입력으로 사용하고, 도입 시 날짜 partition·tenant/device clustering·query 상한을 적용하는 조건

## 결정 예정 항목

- GPS 품질 플래그와 경로 정제 기준선
- 마스킹 거리 또는 시간 기준의 실험 범위
- 원본·receipt·projection별 접근권한, lifecycle, 보존·삭제 정책
- 익명 공간 집계의 최소 기기·사용자 수와 BigQuery GIS 활성화 조건
- Cloud Run·Storage·Firestore 서비스별 비용 지표, budget alert, 운영 상한
- 합성 batch 부하 테스트와 Emulator/Security Rules 회귀 테스트의 통과 기준
- lease 획득·갱신·만료, stale fence mutation 차단, recovery 분류와 cleanup 경쟁의 통과 기준

## 기대 산출물

- 원본/정제/마스킹 경로 비교 지도
- Storage object→Firestore receipt→trip summary/current projection 데이터 계보
- 위치 프라이버시 위협 모델과 lifecycle·삭제 통제표
- 합성 batch 기반 Cloud Run 처리량·복구·서비스별 비용 테스트 결과
- lease/fence 상태 전이표와 stale finalizer 차단 테스트
- 네 가지 crash artifact 상태의 recovery matrix 및 deadline cleanup 경쟁 테스트
- Emulator Suite 기반 tenant 격리와 Firestore/Storage Security Rules 회귀 결과
- 필요성이 확인된 경우에만 BigQuery GIS/ML 비용·지연 PoC와 활성화 판단 기록

## 검증 지표

- 정제 전후 실제 이동거리 왜곡이 허용 범위 내인가
- 소수 이용자 영역이 기관 집계에서 숨겨지는가
- 삭제 요청이 Firestore receipt/projection, Storage object, 활성화된 BigQuery 파생 데이터까지 추적되는가
- 목표 batch 수에서 오류율·p95 처리 지연·재처리 성공률과 서비스별 비용을 함께 측정했는가
- sample별 Firestore write가 발생하지 않고 listener 대상이 작은 current projection으로 제한되는가
- Emulator/Security Rules 회귀 테스트가 tenant 간 접근과 무권한 object 접근을 차단하는가
- 모든 pending receipt에 active lease owner가 하나뿐이고 fencing token이 단조 증가하며 stale fence mutation이 0인가
- raw-only/no-artifact/manifest-only/stored-missing 복구 결과와 cleanup 전환 경쟁이 결정론적으로 재현되는가
