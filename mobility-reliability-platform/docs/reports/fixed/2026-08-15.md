# 2026년 8월 1차 기술 정기리포트

> 문서 상태: 계획 기준 사전작성본  
> 실제 증빙: 회의 후 갱신

## 목적

GPS·motion 특성으로 수집 데이터의 품질과 이동 유형을 보조 판별하는 학습 문제를 정의한다. 모델보다 먼저 라벨, split, baseline, 실패 유형을 고정해 데이터 누수와 과장된 성능을 방지한다.

## 계획상 논의

- 전동보장구 주행, 도보, 차량, 정지, 불확실 구간의 라벨 정의
- 위치 정확도, 속도 분포, 가속·감속, 정지 비율, 방향 변화, 이동거리 대비 변위 특성
- 동일 사용자·동일 경로가 train/test에 섞이지 않도록 group/time split 적용
- 규칙 기반 baseline과 로지스틱 회귀, 작은 1D CNN/GRU 후보 비교
- 불확실 구간을 억지 분류하지 않는 abstain 라벨
- 사용자 동의와 라벨링 데이터의 보존·삭제 원칙
- BigQuery 활성화 전에는 승인된 Cloud Storage manifest를, 활성화 후에는 비용이 기록된 partitioned snapshot만 사용하는 학습 데이터 경계

## 결정 예정 항목

- 모델이 답해야 할 질문과 답하지 않을 질문
- 라벨링 가이드, 검토 절차, disagreement 처리
- 최소 데이터량 이전에 사용할 규칙 기반 fallback
- primary metric과 class imbalance 대응
- 재현 가능한 학습 환경, Storage manifest 또는 optional BigQuery snapshot, 데이터셋 버전 규칙

## 기대 산출물

- 원본 Storage object 계보와 manifest ID를 포함한 데이터셋 카드 초안 및 라벨링 도구 또는 화면
- 규칙 기반 품질 판별 baseline
- PyTorch 학습 파이프라인 골격
- 데이터 split 및 leakage 검사
- 대표 실패 유형 목록과 confusion matrix 템플릿

## 검증 지표

- 두 라벨러 또는 재검토 간 일치도를 기록할 수 있는가
- 사용자·시간 기준 split이 데이터 누수를 차단하는가
- class별 precision/recall과 abstain 비율이 함께 보고되는가
- baseline 오류가 원시 구간과 연결되어 재현 가능한가
