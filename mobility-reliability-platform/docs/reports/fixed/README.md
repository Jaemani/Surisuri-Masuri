# 기술 정기리포트 사전작성본

이 디렉터리는 Mobility Reliability Platform의 2026년 5월~12월 기술 정기리포트 16개를 보관한다. 매월 15일과 말일을 기준으로, 실제 개발 속도와 무관하게 8개월 기술 로드맵의 관점에서 검토할 의제와 기대 증거를 미리 정의한다.

## 사용 원칙

- 모든 문서는 계획 기준 사전작성본이며, 작성된 계획을 실제 성취로 해석하지 않는다.
- 회의 후 실제 진행 입력란, 실제 결정, 차이와 원인, 증빙 링크를 갱신한다.
- 완료 판단은 커밋, 테스트 결과, 화면, 그래프, ADR, 모델 평가물 중 하나 이상의 증빙이 있을 때만 기록한다.
- 계획보다 빠르거나 느린 경우 계획을 거짓으로 맞추지 않고, 차이와 다음 회차의 조정안을 기록한다.
- 기술 성과가 없는 회차에도 가설, 비교 대안, 실패 원인, 측정 결과를 남겨 의사결정의 근거를 보존한다.
- 참석자, 회의 일시, 외부 협의, 실증 사용자 수, 성능 수치는 실제 확인 후에만 입력한다.

각 문서는 `보고서 ID`, `계획 기준일`, `실제 세션·발행일`, `검토 상태`를 분리한다. `실제 세션·발행일: 미입력`은 회의나 발행이 있었다는 뜻이 아니며, 사람 확인 전 과거 회차를 소급 완료 처리하지 않는다.

## 실제 회의 후 갱신 순서

1. 실제 진행 입력란에 수행·미수행·변경 사항을 구분해 적는다.
2. 결정 예정 항목을 실제 결정과 보류 항목으로 나눈다.
3. 측정값에는 데이터 범위, 장비, 버전, 실행 조건을 함께 기록한다.
4. 증빙 링크 입력란에 저장소 상대 경로나 외부 문서의 안정적인 URL을 연결한다.
5. 다음 회차는 현재 결과에 따라 범위를 조정하되, 원래 8개월 게이트와의 차이를 남긴다.
