## 목적

신뢰성 모델의 점수가 실제 위험과 일치하는지 calibration을 검증하고, 데이터 부족이나 분포 이탈 시 결과를 유보하는 정책을 정의한다. 숫자 하나보다 근거·범위·불확실성을 중심으로 UX를 설계한다.

## 계획상 논의

- time horizon별 calibration curve, Brier score, confidence interval
- 부품·기종·기관·주행량 구간별 subgroup 성능
- 점검 권장, 관찰 유지, 데이터 부족, 즉시 전문가 확인의 상태 구분
- fixed threshold와 비용 기반 threshold 비교
- out-of-distribution·누락 feature·낮은 데이터 품질의 abstention 조건
- 사용자용 간단 설명과 수리사용 feature·event 근거의 분리

## 결정 예정 항목

- 서비스에 표시할 위험 구간과 금지할 표현
- 모델 적용 최소 데이터 조건
- calibration 방법과 재평가 주기
- 수리사 피드백이 outcome correction으로 반영되는 절차
- 예측 유보 시 적용할 기본 점검 규칙

## 기대 산출물

- calibration plot과 subgroup 평가표
- threshold/abstention 정책 문서
- 모델 위험 등록부와 배포 승인 체크리스트
- 사용자·수리사 위험 설명 화면
- 모델 결과→feature→원천 이벤트 계보

## 검증 지표

- 전체와 주요 subgroup에서 calibration 오차가 보고되는가
- 예측을 유보한 비율과 그 사유를 분류할 수 있는가
- 누락·비정상 입력에서 규칙 기반 fallback이 동작하는가
- UI가 고장 시점의 확정적 예측으로 오해될 표현을 피하는가
