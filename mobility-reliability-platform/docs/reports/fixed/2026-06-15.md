# 2026년 6월 1차 기술 정기리포트

> 문서 상태: 계획 기준 사전작성본  
> 실제 증빙: 회의 후 갱신

## 목적

Android와 iOS 실기기에서 background GPS 수집의 생명주기와 실패 조건을 검증한다. 단순 성공 화면이 아니라 배터리 소비, 정확도, 강제 종료·권한 변경·재부팅 상황을 포함한 운영 기준을 만든다.

## 계획상 논의

- foreground, background, 화면 잠금, 앱 강제 종료, 기기 재부팅 시 수집 동작
- 플랫폼별 always/while-in-use 권한과 사용자 안내 방식
- 시간·거리·정확도 기반 샘플링 후보와 배터리 비용
- 위치 이벤트의 로컬 append-only 저장 및 중간 유실 방지
- 수집 불가 상태를 사용자와 운영자에게 표시하는 진단 이벤트
- 민감한 출발·도착 지점이 서버에 전송되기 전 적용할 최소 보호 조치

## 결정 예정 항목

- 플랫폼별 지원 가능한 background 수집 보장 수준
- 기본 샘플링 정책과 저전력 모드의 후보 값
- 강제 종료 이후 자동 복구가 불가능한 경우의 UX
- 이벤트 정확도 하한과 제외 대신 품질 플래그를 남길 조건
- 위치 권한 변경을 감사 이벤트로 기록할지 여부

## 기대 산출물

- 실기기 테스트 매트릭스
- 샘플링 정책별 배터리·거리 오차 비교 그래프
- 위치 수집 상태 머신 및 진단 코드 목록
- 권한 거부·회수·복구 화면
- 심각 오류 기록 초안: 수집 유실과 중복의 분류 기준

## 검증 지표

- 각 OS 상태별 수집 성공·실패가 재현 가능한가
- 샘플 수, 기록 간격, GPS accuracy 분포가 자동 집계되는가
- 앱 상태 전환 중 이벤트 ID 연속성과 로컬 보존을 확인했는가
- 사용자에게 실제 보장 수준을 과장하지 않는 안내가 제공되는가
