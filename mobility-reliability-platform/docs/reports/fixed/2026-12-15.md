# 2026년 12월 1차 기술 정기리포트

> 문서 상태: 계획 기준 사전작성본  
> 실제 증빙: 회의 후 갱신

## 목적

8개월 동안 구축한 모바일 텔레메트리, offline sync, 공간 파이프라인, 상태 projection, ML, 생존분석, 근거형 보고서, 기관 콘솔을 하나의 재현 가능한 최종 평가로 검증한다.

## 계획상 논의

- 고정된 앱·서버·데이터·모델·프롬프트 버전으로 release candidate 구성
- 새로운 주행 기록→offline 복구→상태 갱신→위험 설명→수리 기록→재계산의 end-to-end 시나리오
- 수집 누락·중복, 배터리·지연, projection 정합성, 모델 calibration, report groundedness KPI
- 실데이터·합성데이터·demo fixture를 명확히 구분하는 표시
- 알려진 제한과 실패 사례를 포함한 모델·시스템 카드
- 발표용 화려함과 검증 가능한 증거를 함께 보이는 데모 흐름

## 결정 예정 항목

- release candidate와 평가 데이터 동결 기준
- 최종 KPI의 계산식·분모·데이터 범위
- 실패 시 재촬영 대신 원인과 fallback을 보여줄 데모 정책
- 공개 가능한 코드·샘플 데이터·benchmark 범위
- 발표에서 사용할 주장과 근거의 매핑

## 기대 산출물

- end-to-end 평가 실행 스크립트와 결과 패키지
- 핵심 KPI 대시보드 및 benchmark appendix
- 5분 데모 영상 초안과 발표 스토리보드
- 시스템·데이터·모델 카드
- 알려진 제한, 실패 분석, 향후 실증 과제 목록

## 검증 지표

- 깨끗한 환경에서 평가와 데모 fixture가 재현되는가
- 모든 발표 수치가 버전·데이터 범위·계산식과 연결되는가
- 핵심 흐름에서 실패가 발생해도 데이터 유실 없이 복구되는가
- 사용자·수리사·기관 관점의 가치가 동일한 데이터 계보로 설명되는가
