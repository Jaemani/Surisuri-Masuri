# 증거 인덱스 운영

증거는 제품 업데이트, 인시던트, 사람 대상 리포트의 **실제 주장**을 검증할 수 있게 하는 자료다. 이 디렉터리는 설명 없는 파일 보관함이 아니라 증거의 출처·환경·무결성·사용처를 기록하는 인덱스다.

## 증거가 필요한 경우

- 기능이 작동하거나 문제가 해결됐다고 주장할 때
- 성능, 정확도, 배터리, 데이터 품질, 접근성 수치를 보고할 때
- 제품 업데이트 또는 인시던트 복구를 검증할 때
- 현장·테스트·합성 데이터 결과를 구분해 보고할 때
- 회의에서 실제 산출물을 검토했다고 기록할 때

## 허용되는 증거 유형

- Git 커밋·PR·태그와 변경 파일
- 자동 테스트·부하 테스트·모델 평가 결과
- 화면 캡처·짧은 데모 영상·지도 전후 비교
- 배포 로그, 모니터링 그래프, 장애 타임라인
- ADR, 데이터 스키마, 사람이 확인한 체크리스트
- 개인정보를 제거한 현장 피드백·집계 수치

## 식별자와 메타데이터

- ID: `EVD-YYYYMMDD-NNN`
- 권장 인덱스 파일명: `YYYY-MM.md` 또는 별도 증거 묶음 `EVD-...-short-slug.md`
- 각 항목의 필수 메타데이터:
  - 제목과 증거 ID
  - 생성 시각과 생성자
  - 출처 도구·명령·커밋 또는 수동 확인 방법
  - 환경: `local | staging | production | field`
  - 데이터 유형: `synthetic | test | field`
  - 저장 위치와 접근 권한
  - 상태: `generated | reviewed | verified | redacted | superseded`
  - 무엇을 증명하고 무엇을 증명하지 못하는지
  - 이를 참조하는 `UPD`, `INC`, `HR`
  - 가능하면 파일 해시와 재현 명령

예시:

```md
### EVD-20260721-001 — Android 오프라인 이벤트 복구 테스트

- 생성: 2026-07-21 16:20 KST / 작성자
- 환경·데이터: local / test
- 출처: commit `abcdef1`, test run `...`
- 위치: `artifacts/...`
- 상태: verified
- 증명: 앱 재시작 후 테스트 이벤트 100건이 로컬 큐에 유지됨
- 한계: 실제 백그라운드 OS 종료와 현장 네트워크는 검증하지 않음
- 사용처: HR-2026-07-B, UPD-20260721-01
```

## 무결성과 변경

- 다른 문서에서 참조된 원본 증거는 덮어쓰지 않는다. 새 실행은 새 ID를 발급한다.
- 잘못된 증거는 삭제로 숨기지 않고 `superseded`로 바꾸고 대체 ID를 연결한다.
- 표·그래프는 가능하면 원시 집계값, 생성 버전, 재현 방법을 함께 보관한다.
- 수동 확인은 확인자와 확인일을 기록한다. 자동 생성 결과는 사람 검토 전 `generated`다.
- 증거 파일을 저장소 외부에 둘 경우 지속 가능한 링크와 접근권한을 기록한다.

## 개인정보와 보안

- 원본 GPS 좌표, 정확한 자택·복지관 출발지, 이름, 연락처, 기기 고유식별자, 인증 토큰, 비밀키를 저장하지 않는다.
- 지도 화면은 시작·종료 위치를 마스킹하고 소수 사용자 영역을 숨긴다.
- 현장 데이터는 집계·가명처리하고, 필요한 최소 기간만 보관한다.
- 보안 인시던트 증거는 악용 가능한 세부를 공개 문서에서 제거하고 승인된 제한 저장소를 사용한다.
- 회의 사진·영수증은 사람이 승인한 별도 저장소에 두며, 존재하지 않는 파일이나 메타데이터를 생성하지 않는다.

## 상호 링크 규칙

- 제품 업데이트는 관련 증거를 연결한다: [`../product-updates/`](../product-updates/)
- 인시던트는 탐지·복구·재발 방지 증거를 연결한다: [`../incidents/`](../incidents/)
- fixed·human 사람 대상 리포트는 `실제` 주장별 증거를 연결한다: [`../reports/fixed/`](../reports/fixed/), [`../reports/human/`](../reports/human/)
- 중요한 검증 기준의 결정 근거는 [`../decisions/`](../decisions/)에 둔다.
- 증거 항목에도 이를 사용한 `UPD`, `INC`, `HR`, fixed `R` 문서를 역링크하여 고아 증거와 근거 없는 주장을 모두 찾을 수 있게 한다.

## 금지사항

- 설명·출처·환경 없이 파일만 추가하지 않는다.
- 합성 결과를 현장 근거로 재분류하지 않는다.
- 실패한 테스트를 숨기거나 성공한 일부 실행만 선택해 일반화하지 않는다.
- 성능 수치의 기간·모수·단위를 생략하지 않는다.
- 참석자, 사진, 지출 또는 측정값을 자동 생성·추정하지 않는다.
