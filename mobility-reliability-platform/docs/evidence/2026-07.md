PASS valid domain event

첫 실행에서는 동일한 `$id`를 가진 schema를 case마다 다시 compile해 Ajv가 두 번째 case에서 중복 schema 오류를 냈다. schema path별 validator cache를 추가한 뒤 전체 fixture를 다시 실행해 위 결과를 확인했다. 이 오류는 런타임 장애가 아니라 로컬 계약 검사기의 최초 구현 오류이며, 실패 실행을 성공 결과로 숨기지 않는다.

- 증명하지 못함: 실제 모바일 payload, 서버 코드, 네트워크 전송, 데이터베이스 constraint, 멱등성 동작.

- 처음 `pnpm check`, `pnpm test`, `pnpm build`를 병렬 실행해 두 Firebase Emulator Suite가 hub `4400`과 Firestore `8080`을 경쟁했다. 한 Rules 실행은 14건 통과했지만 다른 실행은 port discovery 실패로 종료됐다.

- CI와 같은 순차 방식으로 `pnpm test`를 재실행해 mobile 65건, Rules 14건, contract 6 case가 통과했다.
- 이 실패는 local test orchestration 오류이며 production·field runtime, 사용자 데이터, 배포에는 영향이 없어 Incident 기준에 해당하지 않는다. WSL runbook에 Firebase Emulator 검사 병렬 실행 금지를 추가했다.

### 증명 범위와 한계

- 증명: verifier의 header/parser·principal·allowlist·오류 매핑, SDK wrapper의 fake seam, production factory의 emulator 환경검사, Docker source 포함과 fail-closed 상태가 local synthetic/clean container에서 동작한다.

- local image는 registry에 push하거나 Cloud Run에 배포하지 않았다.
