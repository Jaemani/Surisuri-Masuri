## EVD-20260721-003 — 계약 성공·실패 fixture 검증

- 생성: 2026-07-21 KST / 시각 미기록
- 생성자: Codex 명령 실행
- 환경·데이터: `local` / `synthetic fixture`
- 상태: `generated` — 사람 검토 전
- 대상:
  - `packages/contracts/scripts/validate-fixtures.mjs`
  - `packages/contracts/fixtures/telemetry-batch.valid.json`
  - `packages/contracts/fixtures/telemetry-batch.invalid.json`
  - `packages/contracts/fixtures/domain-event.valid.json`
  - `packages/contracts/fixtures/domain-event.invalid.json`

### 확인 명령

```bash
rtk pnpm test
```

최종 관측 결과:

```text
PASS valid telemetry batch
PASS invalid telemetry batch
PASS valid domain event
PASS invalid domain event
```

첫 실행에서는 동일한 `$id`를 가진 schema를 case마다 다시 compile해 Ajv가 두 번째 case에서 중복 schema 오류를 냈다. schema path별 validator cache를 추가한 뒤 전체 fixture를 다시 실행해 위 결과를 확인했다. 이 오류는 런타임 장애가 아니라 로컬 계약 검사기의 최초 구현 오류이며, 실패 실행을 성공 결과로 숨기지 않는다.

### 증명 범위와 한계

- 증명: Draft 2020-12 validator가 두 schema를 compile하며, 준비된 유효 fixture는 허용하고 의도적으로 잘못된 fixture는 거부한다.
- 증명하지 못함: 실제 모바일 payload, 서버 코드, 네트워크 전송, 데이터베이스 constraint, 멱등성 동작.
- 증명하지 못함: fixture가 모든 경계값과 향후 schema compatibility를 포괄한다는 사실.
