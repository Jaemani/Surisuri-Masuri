# 제품 업데이트: 신규 프로젝트 기반선 생성

## 요약

기존 세 코드베이스를 연장하지 않는 신규 프로젝트 폴더와 8개월 기술 운영을 위한 문서·계약 골격을 생성했다. 이번 업데이트는 개발자용 기반선이며, 사용자에게 제공되는 앱·서버·GPS·데이터베이스·모델·현장 기능의 구현 또는 배포를 포함하지 않는다.

## 변경 전 문제

- 관측된 문제: 신규 제품의 범위, 재사용 경계, 5월~12월 기술 게이트와 기록 방식이 하나의 신규 코드베이스에서 고정되어 있지 않았다.
- 영향을 받은 사용자·업무: 개발 착수 기준, 의사결정 추적, 월 2회 기술 리포트, 서비스 간 데이터 계약 준비.
- 기준선 또는 이전 동작: 과거 저장소와 대화·첨부 자료는 존재했지만 신규 프로젝트의 실행 기준으로 사용할 독립 골격은 없었다.

## 변경 후 동작

- 신규 코드베이스 폴더와 모바일·콘솔·게이트웨이·ML·계약·인프라용 디렉터리 골격이 존재한다.
- 프로젝트 헌장과 5월~12월 로드맵이 신규 프로젝트의 범위와 월별 게이트를 고정한다.
- 7개 ADR이 greenfield 경계, 모바일 GPS 세션, 오프라인 이벤트 동기화, 런타임 경계, 위치·신원 분리, 모델·LLM 책임과 Firebase 우선 운영 구조를 기록한다.
- 의사결정, 제품 업데이트, 심각한 오류, 사람 대상 리포트, 증거의 문서 스트림이 분리되어 있다.
- 텔레메트리 배치와 도메인 이벤트의 JSON Schema 초안 2개가 존재하며 JSON 구문 파싱을 통과했다.
- 2026년 5월부터 12월까지 월 2회, 총 16개의 기술 정기리포트 사전작성본이 존재한다.

## 범위

- 포함:
  - 신규 프로젝트 폴더와 코드베이스 디렉터리 골격
  - `docs/PROJECT_CHARTER.md`, `docs/ROADMAP.md`
  - `docs/decisions/ADR-0001`부터 `ADR-0007`까지 7개 결정문
  - 문서 운영 정책, 스트림별 README와 템플릿
  - JSON Schema 2개
  - `docs/reports/fixed/`의 계획 기준 사전작성본 16개
- 제외:
  - React Native 앱과 Android/iOS 실기기 동작
  - 모바일 GPS, SQLite, 오프라인 동기화
  - Go 게이트웨이 runtime, Firebase 인증 연동, 실제 tenant 격리
  - ML 학습·ONNX 추론·신뢰성 분석·AI 리포트
  - 복지관 콘솔, 데이터 이관, 배포, 현장 실증
- 배포 환경: `local`
- 데이터 유형: 해당 없음 — 실행 데이터가 생성되지 않음

## 배포와 롤백

- 배포 방식·식별자: 런타임 배포 없음. 로컬 저장소의 `repository-foundation-v0` 문서·계약 기반선.
- 기능 플래그 또는 점진 배포: 해당 없음.
- 롤백 조건: 해당 없음. 후속 결정으로 기반선이 바뀌면 새 ADR과 업데이트로 대체한다.
- 롤백 절차: 참조된 기록을 삭제하거나 덮어쓰지 않고 `superseded` 상태와 대체 문서를 연결한다.

## 알려진 제한과 후속 작업

- 제한: 모든 앱·서버·데이터·ML·현장 기능은 미착수다.
- 제한: JSON Schema는 JSON 구문 파싱만 확인했으며 Draft 2020-12 의미 검증과 예제 payload 검증은 수행하지 않았다.
- 제한: 정기리포트 16개는 계획 기준 사전작성본이며 실제 회의나 기술 성취를 증명하지 않는다.
- 후속 작업: 스키마 validator와 계약 예제를 추가하고 첫 모바일 GPS vertical slice의 실제 증거를 생성한다.
- 후속 검증 시점: 해당 구현 착수 후 별도 제품 업데이트에서 기록한다.
