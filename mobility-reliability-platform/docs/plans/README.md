# 상세 실행계획

이 디렉터리는 [프로젝트 헌장](../PROJECT_CHARTER.md)과 [8개월 로드맵](../ROADMAP.md)을 실행·검증 가능한 계획으로 분해한다.

| 문서 | 책임 |
| --- | --- |
| [마스터 실행계획](./MASTER_EXECUTION_PLAN.md) | 작업흐름, 의존성, 5~12월 반월 계획, 완료 정의 |
| [검증·증거 계획](./VALIDATION_AND_EVIDENCE_PLAN.md) | 증거 수준, KPI, 실기기·통합·모델·agent 검증 |
| [정기보고·회의 운영계획](./REPORTING_AND_MEETING_PLAN.md) | 월 2회 fixed report, 실제 회의·증거·시각 산출물 |
| [데이터·ML·AI 실행계획](./DATA_ML_AI_PLAN.md) | 데이터 계보, 품질 모델, 신뢰성 모델, Fact/agent |
| [릴리스·파일럿·최종 데모 계획](./RELEASE_PILOT_DEMO_PLAN.md) | 환경 승격, 현장 운영, 발표·포트폴리오 |
| [위험 등록부](./RISK_REGISTER.md) | 위험 신호, 예방·대응, 승격 차단 조건 |

## 우선순위 충돌 시

1. 개인정보·동의·tenant 경계와 사용자 안전
2. 프로젝트 헌장의 범위·비목표
3. 데이터·runtime 계약과 복구 가능성
4. 월별 기술 게이트
5. 발표 편의와 선택 기능

계획 문서가 현재 코드 상태와 충돌하면 계획을 완료 사실로 사용하지 않는다. 실제 상태는 Product Update와 Evidence를 확인하고 차이는 정기리포트에 기록한다.
